Pusher is a block which is used for moving other blocks in conjunction with Platforms. This is its interface:

On receiving a redstone signal, Pusher will attempt to move all Platforms and blocks adjacent to platforms to the direction it is configured to; it will also move any entities on top of the blocks. For convenience, Pusher sides have letters that identify the facings of the sides.

First, pusher checks whether there are any blocks that prevent the movement to target direction. If there are any, then these blocks will be highlighted in the world, and positions will be printed to the console:

Color of highlight is configurable.

If there are no obstacles, pusher will move the blocks (including those with block entities). Be warned that Pusher performs all its work at once, in a single tick. Although it is optimized, it still can cause lag proportional to the number of moved blocks, and dependent on computational power of your computer. You can set a limit of how many platforms the Pusher can move in the settings.

Technical details

In fact, Platform's and Pipe-Relocator's block entities implement the same interface which allows the Pusher to treat them as "block entities that can move other blocks". In theory, any other block with block entity which implements that interface can be used in such way. It can be exposed in the API if someone wants to create own platform-like blocks.
